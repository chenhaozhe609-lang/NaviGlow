## 目录说明

- `deliverables/`：可分享、可评审、可导出的工作稿和版本归档。
- `information-architecture/`：故事线、页结构、slide-by-slide 内容规划。
- `ref/`：参考样式、竞品 deck、截图材料。
- `temporary/`：临时脚本、浏览器输出、中间截图；已被 Git 忽略。

## 使用规则

- 改 PPT 前先读 `information-architecture/`，不要直接改视觉稿。
- 页面标题必须尽量写成结论，不写成“行业趋势”“方案介绍”这类信息标签。
- 任何 AI 场景页都要回答：为什么做、在哪个消费者旅程节点做、创造什么商业价值、为什么只有 YSL 适合。
- 正式工作稿放 `deliverables/current/`；实验版放 `deliverables/prototypes/`；过期版本放 `deliverables/archive/`。
- 临时渲染输出和浏览器 profile 放 `temporary/`。
