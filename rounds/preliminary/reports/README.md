# Reports

这个目录用于保存后续报告正文、测算、附录和提交前整合稿。

## 当前状态

- 目前暂无正式报告稿。
- 初赛正文主交付仍以 8 页 PPT PDF 为核心。
- 本目录可用于沉淀 PPT 之外的支撑材料，例如财务估算逻辑、附录草稿、答辩备忘或最终提交说明。

## 建议文件类型

- `financial-model-notes.md`：财务或商业量级估算假设。
- `appendix-draft.md`：2-3 页以内附录草稿。
- `final-submission-checklist.md`：提交前检查清单。
- `qa-prep.md`：评委问答准备。

## 使用规则

- 不要把 8 页正文核心观点藏在这里；核心观点必须进入 PPT 主体。
- 财务模型不追求精确到小数点，但必须说明估算口径、关键变量和量级逻辑。
- 附录只放补充证据、测算假设、风险细节或利益方地图。
- 如果报告内容改变 deck 结论，应同步更新 `slides/` 和 `decision-records/`。

## 给下一次 Codex 的接力提示

官方 Q&A 明确：附录不超过 2-3 页，财务模型需要有逻辑支撑但不必极其精确。写这里的内容时要服务 PPT，而不是另起一套报告叙事。
